# Latvian Listening Library Architecture

## Overview

This document describes the technical architecture for the Latvian A1/A2 Listening Library, with emphasis on readiness for AI-powered features, audio streaming, and product analytics.

## Current Stack

- **Frontend**: Static HTML/CSS/JS, served from `web/` directory
- **Audio**: MP3 files stored in `data/A1_klausisanas/audio/` and `data/A2_klausisanas/audio/`
- **Catalog**: JSON file (`web/catalog.json`) generated by `scripts/build_catalog.py`
- **Processing**: Python scripts using Groq Whisper and LibreTranslate
- **Hosting**: Fly.io (static asset serving)

## Audio Delivery Optimization

### Current Behavior

The catalog loads all metadata on page load, but audio only loads when selected.

```javascript
audio.src = source.getAudioUrl(item.audio_url);
audio.preload = "metadata"; // Lazy-loads duration/format info only
```

### Recommended Headers

For CDN deployment, add cache headers:

```
Cache-Control: public, max-age=86400, s-maxage=3600
```

For audio files:

```
Cache-Control: public, max-age=86400
```

## Waveform Pre-generation

### Optional Feature

Generate waveform JSON files alongside audio:

```json
{
  "peaks": [0.1, 0.3, 0.5, 0.2, ...],
  "duration": 45.2,
  "sampleRate": 44100
}
```

### Usage

```javascript
const waveformUrl = audioSource.getWaveformUrl(item.audio_url);
fetch(waveformUrl).then(r => r.json()).then(data => {
  if (data.peaks) renderWaveform(data.peaks);
});
```

## HLS/Streaming Readiness

### Audio Source Abstraction

The `AudioSource` class supports multiple audio types:

```javascript
const source = new AudioSource(AudioSourceType.MP3);
// Future: const source = new AudioSource(AudioSourceType.HLS);

// Get URL based on type
const url = source.getAudioUrl('audio/lesson1.mp3');
```

### Migration Path

1. Generate `.m3u8` playlists and `.ts` segments
2. Switch `AudioSourceType` from `MP3` to `HLS`
3. Update CDN configuration for HLS

No code changes required beyond the type switch.

## Serverless Transcription Pipeline

### Design Outline

Location: `scripts/transcription_pipeline.py`

```python
# Pipeline stages:
# 1. Input: MP3 file from data/ or external source
# 2. Transcribe: Groq Whisper → Latvian transcript
# 3. Segment: Timestamp extraction (word-level)
# 4. Translate: LibreTranslate → English
# 5. Extract: Vocabulary + key phrases
# 6. Emit: lesson.json with all metadata
```

### Example Command

```bash
python scripts/transcription_pipeline.py \
  --input data/A1_klausisanas/audio/1.nodalja.mp3 \
  --output state/output/1.nodalja.json
```

### Pseudocode

```python
def transcribe_pipeline(input_path: str, output_path: str):
    mp3 = load_audio(input_path)

    transcript = whisper_transcribe(mp3, language='lv')
    segments = extract_timestamps(transcript)

    translations = translate_segments(segments, from_lang='lv', to_lang='en')

    vocab = extract_vocabulary(transcript)

    lesson = {
        'audio': input_path,
        'transcript': transcript,
        'segments': segments,
        'translations': translations,
        'vocabulary': vocab,
    }

    write_json(output_path, lesson)
```
